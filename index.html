How I ship

Not just building — owning the whole path to production.

The interesting part isn't the code. It's everything between "it works on my machine" and "it's on a stranger's phone." That part is automated.

01 — Build

Full-stack, one language

TypeScript across the stack: React Native and Expo on mobile, Next.js on the web, Supabase and Python on the backend. Native Kotlin when Android demands it.

02 — Release

Automated release lanes

GitHub Actions CI/CD: push a version tag, the pipeline builds the signed artifact and ships it straight to the Play Store. No laptop in the loop, no manual upload.

03 — Verify

Tested before it ships

Invariant test suites gate every release. Maestro end-to-end flows run on physical Android hardware, and an automated visual-QA pass reviews the UI before each deploy.

04 — Write

I document what breaks

40+ deep-dive articles on production failure modes — the OR in an RLS policy that leaked every row, the analytics gotcha, the agent eval that lies. Lessons, not tutorials.
